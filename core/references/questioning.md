# Adaptive questioning (shared by new-book and discuss-chapter)

GBD gathers context through adaptive conversation, not rigid questionnaires.

1. **Start open.** "What's the book?" / "What does this chapter need to do?"
2. **Follow the energy.** When the author lights up about something, dig there.
3. **Challenge vagueness.** Never accept "it should be gripping" — gripping how?
   Whose POV is gripped? What do they fear losing?
4. **Make it concrete.** "Walk me through the scene as a reader experiences it."
5. **Offer options to think against.** Use AskUserQuestion with 2–4 concrete choices
   so the author reacts rather than inventing from a blank page.
6. **Stop when the decision is made.** Lock it (D-01…) and move on.

## AskUserQuestion patterns

Give real, distinct options — they are thinking aids. Example:

```
header: "POV"
question: "Whose head are we in for the opening?"
options:
  - "Mara — intimate, unreliable, we feel her denial"
  - "The detective — cooler, we see Mara from outside"
  - "Omniscient — we know more than any character"
  - "Let me explain"
```

## Freeform escape hatch

If the author picks "Let me explain" (or types freeform), STOP using AskUserQuestion.
Ask in plain prose, listen, then resume structured questions only if useful.

## What to capture vs. defer

Capture decisions that constrain the work (POV, what changes, what must land).
Park good ideas that don't belong to this unit under "Deferred Ideas" in CONTEXT.md
so nothing is lost without bloating the current scope.
